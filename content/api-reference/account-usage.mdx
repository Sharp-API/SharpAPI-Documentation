# Usage Stats

View detailed API usage statistics for your account, including request counts by endpoint, daily breakdowns, streaming activity, and rate limit hits.

```
GET /api/v1/account/usage
```

<Callout type="info">
This endpoint replaces the former `/me/usage` endpoint. The response format has been updated to use the standard v1 envelope.
</Callout>

## Authentication

Requires API key. **Available to all tiers.**

## Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `period` | string | `month` | Time period for usage data: `today`, `week`, `month`, or `all` |

## Response Fields

### Period Object

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Period identifier: `today`, `week`, `month`, or `all` |
| `start` | string | ISO 8601 start of the reporting period |
| `end` | string | ISO 8601 end of the reporting period |

### Requests Object

| Field | Type | Description |
|-------|------|-------------|
| `total` | number | Total API requests in the period |
| `by_endpoint` | object | Request count broken down by endpoint path |
| `by_day` | array | Daily request counts |

### Requests by_day Item

| Field | Type | Description |
|-------|------|-------------|
| `date` | string | Date in `YYYY-MM-DD` format |
| `count` | number | Total requests on that day |

### Streaming Object

| Field | Type | Description |
|-------|------|-------------|
| `total_connections` | number | Total SSE connections opened in the period |
| `total_messages` | number | Total messages sent across all streams |
| `avg_duration_seconds` | number | Average stream connection duration in seconds |
| `peak_concurrent` | number | Maximum concurrent streams during the period |

### Rate Limit Hits

| Field | Type | Description |
|-------|------|-------------|
| `rate_limit_hits` | number | Number of times rate limiting was triggered (HTTP 429 responses) |

## Period Options

| Period | Description | Data Range |
|--------|-------------|------------|
| `today` | Current day (UTC) | Midnight UTC to now |
| `week` | Current week | Last 7 days |
| `month` | Current billing month | Start of current month to now |
| `all` | All time | Account creation to now |

<Callout type="warning">
The `all` period may return large `by_day` arrays for long-lived accounts. Consider using `month` or `week` for routine monitoring.
</Callout>

## Use Cases

### Monitor for Rate Limit Issues

```python
import requests

response = requests.get(
    'https://api.sharpapi.io/api/v1/account/usage',
    params={'period': 'today'},
    headers={'X-API-Key': 'YOUR_API_KEY'}
)
usage = response.json()['data']

if usage['rate_limit_hits'] > 0:
    print(f"WARNING: {usage['rate_limit_hits']} rate limit hits today")
    print("Consider reducing request frequency or upgrading your tier")
```

### Track Endpoint Distribution

```javascript
const { data } = await fetch(
  'https://api.sharpapi.io/api/v1/account/usage?period=month',
  { headers: { 'X-API-Key': 'YOUR_API_KEY' } }
).then(r => r.json());

const total = data.requests.total;
for (const [endpoint, count] of Object.entries(data.requests.by_endpoint)) {
  const pct = ((count / total) * 100).toFixed(1);
  console.log(`${endpoint}: ${count} (${pct}%)`);
}
```
