# Health Check

Check the operational status of the SharpAPI service, including backend connectivity and data freshness.

```
GET /api/v1/health
```

## Authentication

**No authentication required.** This is a public endpoint.

## Example Requests

<Tabs items={['cURL', 'JavaScript', 'Python']}>
  <Tabs.Tab>
```bash
curl -X GET "https://api.sharpapi.io/api/v1/health"
```
  </Tabs.Tab>
  <Tabs.Tab>
```javascript
const response = await fetch('https://api.sharpapi.io/api/v1/health');
const { data } = await response.json();

if (data.status === 'ok') {
  console.log('SharpAPI is healthy');
} else {
  console.warn(`SharpAPI status: ${data.status}`);
  for (const [name, check] of Object.entries(data.checks)) {
    if (check.status !== 'ok') {
      console.warn(`  ${name}: ${check.status} - ${check.message}`);
    }
  }
}
```
  </Tabs.Tab>
  <Tabs.Tab>
```python
import requests

response = requests.get('https://api.sharpapi.io/api/v1/health')
health = response.json()['data']

print(f"Status: {health['status']}")
print(f"Version: {health['version']}")

for name, check in health['checks'].items():
    icon = 'OK' if check['status'] == 'ok' else 'WARN'
    print(f"  [{icon}] {name}: {check.get('message', 'healthy')}")
```

## Response

### Healthy (200)

```json
{
  "data": {
    "status": "ok",
    "version": "1.4.2",
    "timestamp": "2026-02-08T15:00:00.123Z",
    "checks": {
      "redis": {
        "status": "ok",
        "latency_ms": 2,
        "message": "Connected"
      },
      "data": {
        "status": "ok",
        "last_update": "2026-02-08T14:59:58.456Z",
        "age_seconds": 1.7,
        "message": "Data is fresh"
      }
    }
  },
  "meta": {
    "updated_at": "2026-02-08T15:00:00.123Z"
  }
}
```

### Degraded (200)

When the service is partially functional, the status will be `degraded`:

```json
{
  "data": {
    "status": "degraded",
    "version": "1.4.2",
    "timestamp": "2026-02-08T15:00:00.123Z",
    "checks": {
      "redis": {
        "status": "ok",
        "latency_ms": 5,
        "message": "Connected"
      },
      "data": {
        "status": "degraded",
        "last_update": "2026-02-08T14:55:00.000Z",
        "age_seconds": 300,
        "message": "Data is stale (last update 5m ago)"
      }
    }
  },
  "meta": {
    "updated_at": "2026-02-08T15:00:00.123Z"
  }
}
```

### Down (503)

When the service is unable to function, the status will be `down` and the HTTP status code will be 503:

```json
{
  "data": {
    "status": "down",
    "version": "1.4.2",
    "timestamp": "2026-02-08T15:00:00.123Z",
    "checks": {
      "redis": {
        "status": "down",
        "latency_ms": null,
        "message": "Connection refused"
      },
      "data": {
        "status": "down",
        "last_update": null,
        "age_seconds": null,
        "message": "Unable to read data"
      }
    }
  },
  "meta": {
    "updated_at": "2026-02-08T15:00:00.123Z"
  }
}
```

## Response Fields

### Top-Level Fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Overall system status: `ok`, `degraded`, or `down` |
| `version` | string | Current API version |
| `timestamp` | string | ISO 8601 timestamp of this health check |
| `checks` | object | Individual subsystem health checks |

### Status Values

| Status | HTTP Code | Meaning |
|--------|-----------|---------|
| `ok` | 200 | All systems operational |
| `degraded` | 200 | Functional but with reduced performance or stale data |
| `down` | 503 | Service is unavailable |

### Redis Check

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Redis connectivity status: `ok` or `down` |
| `latency_ms` | number \| null | Round-trip latency to Redis in milliseconds |
| `message` | string | Human-readable status message |

### Data Check

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Data freshness status: `ok`, `degraded`, or `down` |
| `last_update` | string \| null | ISO 8601 timestamp of the last data update |
| `age_seconds` | number \| null | Seconds since the last data update |
| `message` | string | Human-readable status message |

## Status Logic

The overall `status` is determined by the worst individual check:

- **ok** - All checks are `ok`
- **degraded** - At least one check is `degraded`, none are `down`
- **down** - At least one check is `down`

### Data Freshness Thresholds

| Data Age | Status | Description |
|----------|--------|-------------|
| < 30 seconds | `ok` | Data is fresh |
| 30 seconds - 5 minutes | `degraded` | Data is stale but usable |
| > 5 minutes | `down` | Data is too old to be reliable |

## Use Cases

### Monitoring and Alerting

Use the health endpoint in your monitoring stack (Prometheus, Datadog, UptimeRobot, etc.) to detect issues:

```bash
# Simple health check for monitoring scripts
STATUS=$(curl -s https://api.sharpapi.io/api/v1/health | jq -r '.data.status')
if [ "$STATUS" != "ok" ]; then
  echo "ALERT: SharpAPI status is $STATUS"
fi
```

### Pre-Request Health Verification

Check health before making critical requests:

```javascript
async function safeRequest(endpoint) {
  const health = await fetch('https://api.sharpapi.io/api/v1/health')
    .then(r => r.json());

  if (health.data.status === 'down') {
    throw new Error('SharpAPI is currently down');
  }

  if (health.data.status === 'degraded') {
    console.warn('SharpAPI is degraded - data may be stale');
  }

  return fetch(`https://api.sharpapi.io/api/v1${endpoint}`, {
    headers: { 'X-API-Key': 'YOUR_API_KEY' }
  }).then(r => r.json());
}
```

### Uptime Dashboard

```python
import requests
from datetime import datetime

response = requests.get('https://api.sharpapi.io/api/v1/health')
health = response.json()['data']

print(f"SharpAPI Status: {health['status'].upper()}")
print(f"Version: {health['version']}")
print(f"Checked at: {health['timestamp']}")
print()

for name, check in health['checks'].items():
    status_icon = {'ok': 'PASS', 'degraded': 'WARN', 'down': 'FAIL'}
    print(f"  [{status_icon.get(check['status'], '???')}] {name}: {check['message']}")

    if name == 'redis' and check.get('latency_ms') is not None:
        print(f"        Latency: {check['latency_ms']}ms")
    if name == 'data' and check.get('age_seconds') is not None:
        print(f"        Data age: {check['age_seconds']}s")
```
