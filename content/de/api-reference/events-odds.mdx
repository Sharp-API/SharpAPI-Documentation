# Event-Quoten

Ruft alle Quoten für ein bestimmtes Event ab, mit optionaler Filterung nach Sportsbook und Markttyp. Gibt eine paginierte Liste von Quoteneinträgen zurück.

```
GET /api/v1/events/{eventId}/odds
```

<Callout type="info">
Dies ist ein Convenience-Endpoint, der `GET /api/v1/odds?event={eventId}` entspricht. Beide liefern dieselben Daten; verwenden Sie denjenigen, der zu Ihrer URL-Struktur passt.
</Callout>

## Authentifizierung

Erfordert einen API key über den `X-API-Key`-Header, den `Authorization: Bearer`-Header oder den `api_key`-Query-Parameter. Verfügbar für alle Tarife.

## Pfadparameter

| Parameter | Typ | Erforderlich | Beschreibung |
|-----------|------|----------|-------------|
| `eventId` | string | Ja | Eindeutige Event-Kennung (z. B. `evt_nba_bos_lal_20260208`) |

## Query-Parameter

| Parameter | Typ | Standard | Beschreibung |
|-----------|------|---------|-------------|
| `sportsbook` | string | all | Filtern nach Sportsbook. Kommagetrennt für mehrere (z. B. `draftkings,pinnacle`) |
| `market` | string | all | Filtern nach Markttyp. Kommagetrennt für mehrere (z. B. `moneyline,spread`) |
| `limit` | integer | 50 | Ergebnisse pro Seite (max. 200) |
| `offset` | integer | 0 | Paginierungs-Offset |

## Response-Header

Alle Antworten enthalten Standard-Header für Rate Limit und Metadaten:

| Header | Beschreibung |
|--------|-------------|
| `X-RateLimit-Limit` | Maximale Anzahl an Anfragen pro Minute für Ihren Tarif |
| `X-RateLimit-Remaining` | Verbleibende Anfragen im aktuellen Zeitfenster |
| `X-RateLimit-Reset` | Unix-Zeitstempel, wann das Rate-Limit-Fenster zurückgesetzt wird |
| `X-Data-Delay` | Datenverzögerung für Ihren Tarif (z. B. `0s`, `60s`) |
| `X-Request-Id` | Eindeutige Anfrage-Kennung zur Fehlersuche |

## Quoten-Objekt

| Feld | Typ | Beschreibung |
|-------|------|-------------|
| `id` | string | Eindeutige Quoten-Kennung |
| `sportsbook` | string | Sportsbook-Kennung (z. B. `draftkings`, `pinnacle`) |
| `event_id` | string | Übergeordnete Event-Kennung |
| `market_type` | string | Markttyp: `moneyline`, `spread`, `total`, `player_prop` usw. |
| `selection` | string | Die Auswahl (Teamname, Spielername, Over/Under) |
| `selection_type` | string | `home`, `away`, `over`, `under` |
| `line` | number \| null | Linienwert für Spread-/Total-Märkte, `null` bei Moneyline |
| `odds.american` | number | American Odds (z. B. -110, +150) |
| `odds.decimal` | number | Dezimalquoten (z. B. 1.909) |
| `odds.probability` | number | Implizite Wahrscheinlichkeit (z. B. 0.5238) |
| `last_seen_at` | string | ISO-8601-Zeitstempel, wann unsere Pipeline diese Zeile zuletzt beobachtet hat. Verwenden Sie diesen als Aktualitätssignal der Pipeline. |
| `odds_changed_at` | string | ISO-8601-Zeitstempel, wann sich der Preis, die Linie oder das `is_live`-Flag zuletzt tatsächlich geändert hat. Vom Sportsbook bereitgestellt, sofern verfügbar; bei Pinnacle wird er über unveränderte Refreshes hinweg fortgeschrieben — siehe [Pinnacles `odds_changed_at` verstehen](/de/concepts/pinnacle-odds-changed-at). |

### Durch alle Quoten paginieren

<Tabs items={['cURL', 'JavaScript', 'Python']}>
  <Tabs.Tab>
```bash
# First page
curl -X GET "https://api.sharpapi.io/api/v1/events/evt_nba_bos_lal_20260208/odds?limit=50&offset=0" \
  -H "X-API-Key: YOUR_API_KEY"

# Next page
curl -X GET "https://api.sharpapi.io/api/v1/events/evt_nba_bos_lal_20260208/odds?limit=50&offset=50" \
  -H "X-API-Key: YOUR_API_KEY"
```
  </Tabs.Tab>
  <Tabs.Tab>
```javascript
const eventId = 'evt_nba_bos_lal_20260208';
let offset = 0;
const limit = 50;
let allOdds = [];

while (true) {
  const response = await fetch(
    `https://api.sharpapi.io/api/v1/events/${eventId}/odds?limit=${limit}&offset=${offset}`,
    { headers: { 'X-API-Key': 'YOUR_API_KEY' } }
  );
  const { data, meta } = await response.json();
  allOdds = allOdds.concat(data);

  if (!meta.pagination.has_more) break;
  offset = meta.pagination.next_offset;
}

console.log(`Fetched all ${allOdds.length} odds`);
```
  </Tabs.Tab>
  <Tabs.Tab>
```python
import requests

event_id = 'evt_nba_bos_lal_20260208'
all_odds = []
offset = 0
limit = 50

while True:
    response = requests.get(
        f'https://api.sharpapi.io/api/v1/events/{event_id}/odds',
        params={'limit': limit, 'offset': offset},
        headers={'X-API-Key': 'YOUR_API_KEY'}
    )
    result = response.json()
    all_odds.extend(result['data'])

    if not result['meta']['pagination']['has_more']:
        break
    offset = result['meta']['pagination']['next_offset']

print(f"Fetched all {len(all_odds)} odds")
```
