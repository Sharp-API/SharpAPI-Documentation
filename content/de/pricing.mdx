## Rate Limiting

Alle API-Anfragen werden pro API-Schlüssel rate-limitiert. Rate-Limit-Header werden in jeder Antwort mitgeliefert:

```http
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 299
X-RateLimit-Reset: 1703123460
```

### Rate-Limit-Antwort

Wenn Sie Ihr Rate Limit überschreiten, erhalten Sie eine `429`-Antwort:

```json
{
  "error": {
    "code": "rate_limited",
    "message": "Rate limit exceeded",
    "retry_after": 45
  }
}
```

<Callout type="info">
Verwenden Sie den `retry_after`-Wert (in Sekunden), um exponentielles Backoff zu implementieren.
</Callout>

## Datenverzögerung

- **Free-Stufe**: 60 Sekunden Verzögerung bei allen Quotendaten
- **Bezahlte Stufen**: Echtzeit-Daten mit Latenz unter einer Sekunde
