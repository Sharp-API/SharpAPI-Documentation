## Rate Limiting

Todas las solicitudes a la API tienen un rate limit por API key. Las cabeceras de rate limit se incluyen en cada respuesta:

```http
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 299
X-RateLimit-Reset: 1703123460
```

### Respuesta al superar el rate limit

Cuando superes tu rate limit, recibirás una respuesta `429`:

```json
{
  "error": {
    "code": "rate_limited",
    "message": "Rate limit exceeded",
    "retry_after": 45
  }
}
```

<Callout type="info">
Utiliza el valor `retry_after` (en segundos) para implementar un backoff exponencial.
</Callout>

## Retraso de datos

- **Plan Free**: retraso de 60 segundos en todos los datos de cuotas
- **Planes de pago**: datos en tiempo real con latencia inferior al segundo
