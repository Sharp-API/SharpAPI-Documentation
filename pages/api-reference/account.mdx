# Account Info

Retrieve your account details, tier information, rate limits, and enabled features.

```
GET /api/v1/account
```

<Callout type="info">
This endpoint replaces the former `/me` endpoint. The response format has been updated to use the standard v1 envelope.
</Callout>

## Authentication

Requires API key. **Available to all tiers.**

## Example Requests

<Tabs items={['cURL', 'JavaScript', 'Python']}>
  <Tabs.Tab>
```bash
curl -X GET "https://api.sharpapi.io/api/v1/account" \
  -H "X-API-Key: YOUR_API_KEY"
```
  </Tabs.Tab>

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `key_id` | string | Identifier for the API key used in this request |
| `tier` | string | Subscription tier: `free`, `hobby`, `pro`, `sharp`, or `enterprise` |
| `status` | string | Account status: `active`, `suspended`, or `cancelled` |
| `created_at` | string | ISO 8601 timestamp of account creation |

### Limits Object

| Field | Type | Description |
|-------|------|-------------|
| `requests_per_minute` | number | Maximum API requests allowed per minute |
| `remaining` | number | Requests remaining in the current rate limit window |
| `resets_at` | string | ISO 8601 timestamp when the rate limit window resets |

### Streaming Object

| Field | Type | Description |
|-------|------|-------------|
| `enabled` | boolean | Whether streaming (SSE) is enabled for this account |
| `max_concurrent` | number | Maximum concurrent SSE streams allowed |
| `active_streams` | number | Number of currently active streams |

### Features Object

| Field | Type | Description |
|-------|------|-------------|
| `odds` | boolean | Access to odds endpoints |
| `schedule` | boolean | Access to schedule/events endpoints |
| `best_odds` | boolean | Access to best odds comparison |
| `ev` | boolean | Access to +EV opportunity detection |
| `arbitrage` | boolean | Access to arbitrage detection |
| `middles` | boolean | Access to middles detection |
| `player_props` | boolean | Access to player prop markets |
| `csv_export` | boolean | Ability to export data as CSV |
| `streaming` | boolean | Access to SSE streaming |
| `priority_support` | boolean | Priority support access |

### Sportsbooks Object

| Field | Type | Description |
|-------|------|-------------|
| `allowed` | number | Number of sportsbooks accessible on your tier |
| `list` | string[] | Array of sportsbook identifiers you can access |

## Use Cases

### Check Feature Access Before Making Requests

```javascript
const { data: account } = await fetch(
  'https://api.sharpapi.io/api/v1/account',
  { headers: { 'X-API-Key': 'YOUR_API_KEY' } }
).then(r => r.json());

if (!account.features.ev) {
  console.log('Upgrade to Pro tier for +EV detection');
} else {
  // Proceed with EV endpoint
  const evData = await fetch(
    'https://api.sharpapi.io/api/v1/opportunities/ev',
    { headers: { 'X-API-Key': 'YOUR_API_KEY' } }
  ).then(r => r.json());
}
```

### Monitor Rate Limit Usage

```python
import requests

response = requests.get(
    'https://api.sharpapi.io/api/v1/account',
    headers={'X-API-Key': 'YOUR_API_KEY'}
)
account = response.json()['data']

usage_pct = (1 - account['limits']['remaining'] / account['limits']['requests_per_minute']) * 100
print(f"Rate limit usage: {usage_pct:.1f}%")
print(f"Resets at: {account['limits']['resets_at']}")
```
