# Markets

Get available market types, their configurations, and sport availability.

```
GET /api/v1/markets
```

<Callout type="info">
This is a **public endpoint** -- no authentication required. Unauthenticated requests are rate-limited to 10 requests/minute. Authenticated requests receive your tier's full rate limit.
</Callout>

## Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `sport` | string | all | Filter markets by sport (e.g., `basketball`) |
| `category` | string | all | Filter by category (e.g., `main`, `prop`, `period`) |

## Example Requests

## Market Object Schema

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique market identifier (use in API filters) |
| `name` | string | Market name (lowercase) |
| `display_name` | string | Human-readable display name |
| `description` | string | Brief description of the market |
| `category` | string | Market category: `main`, `prop`, or `period` |
| `has_line` | boolean | Whether this market includes a line value |
| `selection_types` | string[] | Possible selection types for this market |
| `sports` | string[] | Sport IDs that support this market |

## Market Categories

### Main Markets

Core betting markets available for most events. These are the most popular and widely available.

| Market | Description | Line | Selections |
|--------|-------------|------|------------|
| `moneyline` | Pick the winner | No | `home`, `away` |
| `spread` | Point spread / handicap betting | Yes | `home`, `away` |
| `total` | Over/Under combined score | Yes | `over`, `under` |

### Prop Markets

Player, team, and game proposition bets for more specific outcomes.

| Market | Description | Line | Selections |
|--------|-------------|------|------------|
| `player_prop` | Individual player statistics (points, rebounds, assists, etc.) | Yes | `over`, `under` |
| `team_prop` | Team-specific outcomes (team total points, first to score, etc.) | Yes | `over`, `under`, `yes`, `no` |
| `game_prop` | Game-specific outcomes (will there be overtime, etc.) | No | `yes`, `no` |

### Period Markets

Markets for specific periods within a game (halves, quarters).

| Market | Description | Line | Selections |
|--------|-------------|------|------------|
| `first_half` | 1H moneyline, spread, or total | Yes | `home`, `away`, `over`, `under` |
| `first_quarter` | 1Q moneyline, spread, or total | Yes | `home`, `away`, `over`, `under` |
| `second_half` | 2H moneyline, spread, or total | Yes | `home`, `away`, `over`, `under` |

## Selection Types

| Type | Used In | Description |
|------|---------|-------------|
| `home` | Moneyline, Spread, Period | Home team wins or covers |
| `away` | Moneyline, Spread, Period | Away team wins or covers |
| `over` | Total, Props, Period | Over the line value |
| `under` | Total, Props, Period | Under the line value |
| `yes` | Game Props, Team Props | Outcome occurs |
| `no` | Game Props, Team Props | Outcome does not occur |

## Using Markets in Filters

Use the market `id` as a filter parameter on other endpoints:

```bash
# Get only moneyline odds
curl "https://api.sharpapi.io/api/v1/odds?market=moneyline" \
  -H "X-API-Key: YOUR_API_KEY"

# Get spread and total odds
curl "https://api.sharpapi.io/api/v1/odds?market=spread,total" \
  -H "X-API-Key: YOUR_API_KEY"

# Get player props only
curl "https://api.sharpapi.io/api/v1/odds?market=player_prop" \
  -H "X-API-Key: YOUR_API_KEY"

# Best odds for moneyline markets
curl "https://api.sharpapi.io/api/v1/odds/best?market=moneyline" \
  -H "X-API-Key: YOUR_API_KEY"
```

## Understanding Lines

Markets with `has_line: true` include a numerical line value in the odds response:

- **Spread**: The point handicap (e.g., `-6.5` means the team must win by 7+)
- **Total**: The combined score threshold (e.g., `215.5` for over/under)
- **Player Props**: The stat threshold (e.g., `24.5` points for a player)
- **Team Props**: The team stat threshold (e.g., `105.5` team total points)

Markets with `has_line: false` (like moneyline) will have `line: null` in the odds response.
