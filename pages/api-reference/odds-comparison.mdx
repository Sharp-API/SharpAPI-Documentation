# Odds Comparison

Compare odds for the same selection across multiple sportsbooks. Results are grouped by event, market, and selection so you can see every book's odds side by side in a single response.

```
GET /api/v1/odds/comparison
```

## Authentication

Requires API key. Available to **all tiers**.

<Callout type="info">
The sportsbooks included in the comparison depend on your tier's book access. Free tier compares DraftKings and FanDuel; higher tiers include more books. See [Book Access by Tier](/api-reference/odds#book-access-by-tier).
</Callout>

## Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `sport` | string | all | Filter by sport (e.g., `basketball`) |
| `league` | string | all | Filter by league (e.g., `nba`) |
| `market` | string | all | Filter by market type (e.g., `moneyline`, `spread`) |
| `event` | string | - | Filter by event ID |
| `sportsbook` | string | all | Comma-separated sportsbook IDs to compare |
| `limit` | integer | 50 | Max comparison groups per page (max 200) |
| `offset` | integer | 0 | Pagination offset |

<Callout type="warning">
For best results, use the `event` parameter to compare odds for a specific event. Without it, results are paginated across all events and can be large.
</Callout>

## Comparison Group Schema

Each item in the `data` array represents one selection (e.g., "PHO Suns moneyline") with all available books:

| Field | Type | Description |
|-------|------|-------------|
| `event_id` | string | Event identifier |
| `event_name` | string | Event display name |
| `sport` | string | Sport slug |
| `league` | string | League slug |
| `home_team` | string | Home team name |
| `away_team` | string | Away team name |
| `market_type` | string | Market type |
| `selection` | string | Selection name |
| `selection_type` | string | `home`, `away`, `over`, `under` |
| `line` | number \| null | Line value (for spreads/totals) |
| `books` | array | Array of sportsbook odds for this selection |
| `best_book` | string | Sportsbook ID with the best odds |
| `book_count` | integer | Number of books with odds for this selection |
| `spread` | number | Difference in American odds between best and worst book |
| `event_start_time` | string | ISO 8601 event start time |

### Book Object

Each entry in the `books` array:

| Field | Type | Description |
|-------|------|-------------|
| `sportsbook` | string | Sportsbook ID |
| `odds` | object | Odds in all three formats |
| `odds.american` | number | American odds |
| `odds.decimal` | number | Decimal odds |
| `odds.probability` | number | Implied probability |
| `line` | number \| null | Line at this sportsbook |
| `is_best` | boolean | Whether this is the best available odds |
| `is_sharp` | boolean | Whether this is a sharp book (e.g., Pinnacle) |
| `timestamp` | string | When this book's odds were last updated |

## Understanding the `spread` Field

The `spread` field shows the difference in American odds between the best and worst sportsbook for a given selection. A higher spread indicates more variance across books, which often signals line shopping opportunities.

| Spread | Interpretation |
|--------|---------------|
| 0-5 | Books are tightly aligned |
| 5-15 | Normal variance, moderate shopping value |
| 15+ | Significant discrepancy, strong shopping value |

## Use Cases

### Line Shopping

Find the best price before placing a bet by comparing all books at once:

```bash
curl "https://api.sharpapi.io/api/v1/odds/comparison?event=33483153&market=spread" \
  -H "X-API-Key: YOUR_API_KEY"
```

### Identifying Stale Lines

Look for books that haven't updated recently by checking each book's `timestamp`. A book with stale odds may be slow to adjust, creating temporary value.

### Tracking Consensus

When most books agree on a line but one is an outlier, the outlier may be mispriced or may have received sharp action first.
